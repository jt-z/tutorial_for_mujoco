# MuJoCo UR5e 机械臂项目基础设置详解

## 📐 坐标系统设置

### 1. 世界坐标系（World Frame）

MuJoCo使用**右手坐标系**，原点位于机器人基座位置。

```
        Z轴 ↑ (垂直向上)
            |
            |
            |
            └────────→ Y轴 (机械臂前方)
           /
          /
         ↙ X轴 (机械臂左侧)
```

**可视化标记**（scene.xml中定义）：
- 红色圆柱：X轴正方向（0.3m长）
- 绿色圆柱：Y轴正方向（0.3m长）
- 蓝色圆柱：Z轴正方向（0.3m长）

**坐标系约定：**
- **X轴**：向左为正（红色）
- **Y轴**：向前为正（绿色）
- **Z轴**：向上为正（蓝色）
- **原点**：机器人基座中心，地面上方

## 🤖 UR5e 机械臂关节系统

### 关节配置（6自由度）

```python
start_joints = np.array([-1.57, -1.34, 2.65, -1.3, 1.55, 0])
```

UR5e是一个**6自由度（6-DOF）**串联机械臂，每个关节都是旋转关节。

#### 关节详细说明

| 索引 | 关节名称 | 旋转轴 | 初始角度 | 角度范围 | 功能 |
|------|---------|--------|---------|----------|------|
| 0 | `shoulder_pan_joint` | Z轴 | -1.57 rad (-90°) | ±180° | 基座旋转（水平转动） |
| 1 | `shoulder_lift_joint` | Y轴 | -1.34 rad (-77°) | ±180° | 肩部升降（前后摆动） |
| 2 | `elbow_joint` | Y轴 | 2.65 rad (152°) | ±180° | 肘部弯曲 |
| 3 | `wrist_1_joint` | Y轴 | -1.3 rad (-74°) | ±180° | 腕部1（上下摆动） |
| 4 | `wrist_2_joint` | Z轴 | 1.55 rad (89°) | ±180° | 腕部2（左右转动） |
| 5 | `wrist_3_joint` | Y轴 | 0 rad (0°) | ±180° | 腕部3（末端旋转） |

**注意：**
- 角度单位：弧度（radians），1弧度 ≈ 57.3°
- 旋转方向：遵循右手定则
- `data.qpos[:6]`：读取/设置这6个关节的位置

### 关节角度与机械臂姿态对应

```python
start_joints = [-1.57, -1.34, 2.65, -1.3, 1.55, 0]
# 对应末端执行器位姿：
# 位置: [-0.13, 0.3, 0.1] (单位: 米)
# 姿态: [3.14, 0, 1.57] (欧拉角，单位: 弧度)
```

**初始姿态解读：**
- 关节0 (-90°)：基座向左旋转90度
- 关节1 (-77°)：肩部向后倾斜77度
- 关节2 (152°)：肘部大幅弯曲（手臂收拢）
- 关节3 (-74°)：腕部向下倾斜
- 关节4 (89°)：腕部几乎垂直
- 关节5 (0°)：末端无额外旋转

**视觉效果：**
```
        机械臂侧视图（初始姿态）
        
基座    肩部      肘部
 |       /        /
 |      /        /
 └─────┘        /
        \      /
         \    /  腕部
          \  /   /
           \/___/
              末端 (y=0.3, z=0.1)
```

## 📍 末端执行器位置与姿态

### 1. 位置坐标（Position）

位置使用**笛卡尔坐标系**表示：

```python
# 初始位置
initial_pos = [-0.13, 0.3, 0.1]  # [x, y, z] 单位: 米

# A位置（接近立方体）
target_A_pos = [-0.13, 0.5, 0.1]

# B位置（推动立方体）
target_B_pos = [-0.13, 0.65, 0.1]
```

#### 位置坐标详解

| 坐标 | 初始值 | A位置 | B位置 | 说明 |
|------|--------|-------|-------|------|
| **X** | -0.13m | -0.13m | -0.13m | 末端在机器人左侧13cm |
| **Y** | 0.3m | 0.5m | 0.65m | 逐渐向前推进 |
| **Z** | 0.1m | 0.1m | 0.1m | 保持在地面上方10cm |

**运动轨迹：**
```
俯视图（XY平面，Z=0.1m）

    Y轴 ↑
        |
        |  B位置 ● (y=0.65)
        |    ↑
        |    | 15cm推动
        |    |
        |  A位置 ● (y=0.5)
        |    ↑
        |    | 20cm接近
        |    |
        |  初始 ● (y=0.3)
        |
        |
        └──────────→ X轴
      x=-0.13m
```

### 2. 姿态角度（Orientation）

姿态使用**欧拉角（Euler Angles）**表示，旋转顺序为**XYZ**：

```python
ee_euler = [3.14, 0, 1.57]  # [roll, pitch, yaw] 单位: 弧度
```

#### 欧拉角详解

| 角度 | 值（弧度） | 值（度） | 旋转轴 | 含义 |
|------|-----------|---------|--------|------|
| **Roll** | 3.14 | 180° | X轴 | 末端翻转180° |
| **Pitch** | 0 | 0° | Y轴 | 无俯仰 |
| **Yaw** | 1.57 | 90° | Z轴 | 水平旋转90° |

**姿态解读：**
1. **Roll = 180°**：末端执行器翻转，力传感器朝下
2. **Pitch = 0°**：无前后倾斜
3. **Yaw = 90°**：相对基座坐标系旋转90°

**视觉表示：**
```
初始姿态（末端朝向）:

侧视图:
    末端执行器
        |
        ↓ (Roll=180°翻转后，朝下)
   ●────────  力传感器绿球
   
俯视图:
        Y轴
         ↑
         |
    末端朝向 (Yaw=90°后)
         →
```

### 3. 姿态表示转换

```python
ee_orientation = tf.euler.euler2mat(*ee_euler)
```

**transforms3d库**将欧拉角转换为**旋转矩阵**（3×3矩阵）：

```python
# 欧拉角 [3.14, 0, 1.57] 转换为旋转矩阵:
[[ 0,  0,  1],    # 第1列：新X轴在世界坐标系的方向
 [ 1,  0,  0],    # 第2列：新Y轴在世界坐标系的方向
 [ 0, -1,  0]]    # 第3列：新Z轴在世界坐标系的方向
```

**为什么需要转换？**
- **ikpy库**的IK求解器需要旋转矩阵作为输入
- 旋转矩阵是姿态的标准数学表示
- 避免欧拉角的万向锁（Gimbal Lock）问题

## 🎯 目标点设置详解

### 位置设置策略

```python
target_A_pos = [-0.13, 0.5, 0.1]   # A位置：靠近立方体但不接触
target_B_pos = [-0.13, 0.65, 0.1]  # B位置：推动立方体
```

#### 为什么选择这些坐标？

**1. X坐标保持不变 (x = -0.13m)**
- 理由：与红色立方体X坐标（-0.1m）接近
- 效果：末端在立方体旁边移动，不会错过目标
- 偏移：3cm的偏移（-0.13 vs -0.1）留出安全距离

**2. Y坐标逐步增加**
- **初始**：y = 0.3m（起始位置）
- **A位置**：y = 0.5m（红色立方体的Y坐标）
- **B位置**：y = 0.65m（推过立方体）

| 阶段 | Y坐标 | 距离立方体 | 目的 |
|------|-------|-----------|------|
| 初始 | 0.3m | 后方20cm | 安全起始位置 |
| A位置 | 0.5m | 并列 | 接近准备接触 |
| B位置 | 0.65m | 前方15cm | 推动立方体 |

**3. Z坐标保持恒定 (z = 0.1m)**
- 理由：与红色立方体Z坐标（0.1m）相同
- 效果：水平推动，高度对齐
- 高度：地面上方10cm，立方体中心高度

### 红色立方体位置

```xml
<!-- scene.xml 中定义 -->
<body name="red_box" pos="-0.1 0.5 0.1" euler="0 0 45">
```

**立方体坐标：**
- X: -0.1m（机器人左侧10cm）
- Y: 0.5m（前方50cm）
- Z: 0.1m（地面上方10cm）
- 尺寸：0.2×0.2×0.2m（边长20cm）
- 旋转：绕Z轴旋转45°

**空间关系图：**
```
俯视图 (Z=0.1m平面):

    Y轴 ↑
        |
        |
   B位置 ●━━━━━━━━━━━┓
   y=0.65            ┃
        |            ┃ 推动距离15cm
        |            ┃
   立方体 ■━━━━━━━━━━━┛
   y=0.5, x=-0.1
        | \
        |  \ 3cm偏移
        |   \
   A位置 ●━━━┛
   y=0.5, x=-0.13
        |
        |
   初始 ●
   y=0.3
        |
   ─────┼─────────→ X轴
     x=-0.13
```

## 🔄 逆运动学（IK）参考位置

```python
ref_pos = [0, 0, -1.57, -1.34, 2.65, -1.3, 1.55, 0, 0]
```

### 什么是参考位置？

**ref_pos**是传递给IK求解器的**初始猜测值**，用于加速收敛并避免奇异解。

#### 结构解析

| 索引 | 值 | 含义 |
|------|---|------|
| 0 | 0 | ikpy虚拟基座链节1（固定） |
| 1 | 0 | ikpy虚拟基座链节2（固定） |
| 2-7 | [-1.57, -1.34, 2.65, -1.3, 1.55, 0] | **6个实际关节角度** |
| 8 | 0 | ikpy虚拟末端链节（固定） |

**总长度：9**
- URDF中有9个链节（包括虚拟链节）
- 但只有6个是可控的旋转关节

### 为什么使用start_joints作为参考？

```python
ref_pos = [0, 0] + list(start_joints) + [0]
#         虚拟基座  +  6个关节角度  + 虚拟末端
```

**优势：**
1. **加速收敛**：从当前姿态开始搜索，比随机初始值快
2. **避免突变**：确保解在当前配置附近，运动连续
3. **规避奇异**：避免IK求解器陷入奇异配置（如关节完全伸直）

**IK求解过程：**
```python
joint_angles = my_chain.inverse_kinematics(
    target_A_pos,          # 目标位置 [x, y, z]
    ee_orientation,        # 目标姿态（旋转矩阵）
    "all",                 # 优化所有可控关节
    initial_position=ref_pos  # 从这个配置开始搜索
)

# 返回值：9个元素的数组
# 提取中间6个实际关节角度
target_A_joints = joint_angles[2:-1]
```

## 🧮 数据结构映射

### MuJoCo数据结构

```python
model = mujoco.MjModel.from_xml_path('scene.xml')
data = mujoco.MjData(model)
```

#### 关键数据成员

| 数据成员 | 维度 | 含义 | 本项目使用 |
|---------|------|------|-----------|
| `data.qpos` | [nq] | 广义位置坐标 | `[:6]` = 6个关节角度<br>`[6:13]` = 立方体位姿(7) |
| `data.qvel` | [nv] | 广义速度 | `[:6]` = 6个关节速度 |
| `data.ctrl` | [nu] | 控制输入 | `[:6]` = 6个目标关节角度 |
| `data.sensordata` | [nsensor] | 传感器数据 | `[:3]` = 力传感器(fx, fy, fz) |
| `data.time` | 标量 | 仿真时间 | 当前时间（秒） |

#### qpos详细结构

```python
# UR5e机械臂 (6个关节)
data.qpos[0]   # shoulder_pan_joint
data.qpos[1]   # shoulder_lift_joint
data.qpos[2]   # elbow_joint
data.qpos[3]   # wrist_1_joint
data.qpos[4]   # wrist_2_joint
data.qpos[5]   # wrist_3_joint

# 红色立方体 (7个自由度：位置3 + 四元数4)
data.qpos[6]   # 立方体 x 位置
data.qpos[7]   # 立方体 y 位置
data.qpos[8]   # 立方体 z 位置
data.qpos[9]   # 四元数 w
data.qpos[10]  # 四元数 x
data.qpos[11]  # 四元数 y
data.qpos[12]  # 四元数 z
```

### 控制流程

```python
# 1. 设置目标关节角度
data.ctrl[:6] = target_joints  # 发送控制指令

# 2. 物理仿真步进
mujoco.mj_step(model, data)   # 前进一个时间步（默认0.002秒）

# 3. 读取当前状态
current_joints = data.qpos[:6]   # 读取实际关节角度
current_force = data.sensordata[:3]  # 读取接触力
```

**控制器类型：**
- 本项目使用**位置PD控制器**（在ur5e.xml中定义）
- 控制律：`τ = Kp*(ctrl - qpos) + Kd*(0 - qvel)`
- Kp = 2000（位置增益），Kd = 2000（速度增益）

## 📏 单位系统

### 长度单位：米（m）

| 量 | 单位 | 示例 |
|---|------|------|
| 位置 | 米 (m) | `target_A_pos = [-0.13, 0.5, 0.1]` |
| 速度 | 米/秒 (m/s) | 关节线速度 |
| 加速度 | 米/秒² (m/s²) | 重力加速度 = 9.81 |

### 角度单位：弧度（rad）

| 量 | 单位 | 示例 | 度数等价 |
|---|------|------|----------|
| 关节角度 | 弧度 (rad) | `-1.57` | -90° |
| 角速度 | 弧度/秒 (rad/s) | 关节转速 | |
| 欧拉角 | 弧度 (rad) | `[3.14, 0, 1.57]` | [180°, 0°, 90°] |

**转换公式：**
```python
弧度 = 角度 * π / 180
角度 = 弧度 * 180 / π

# 示例
-1.57 rad = -1.57 * 180 / π ≈ -90°
3.14 rad = 3.14 * 180 / π ≈ 180°
```

### 力与质量单位

| 量 | 单位 | 示例 |
|---|------|------|
| 力 | 牛顿 (N) | `data.sensordata[:3]` |
| 质量 | 千克 (kg) | 立方体质量 = 1.0 kg |
| 转矩 | 牛顿·米 (N·m) | 关节输出力矩 |

## 🔍 常见问题解答

### Q1: 为什么X坐标是负数？

**A:** 根据右手坐标系定义：
- X轴正方向：机器人右侧
- X轴负方向：机器人左侧
- `x = -0.13m` 表示目标在机器人左侧13cm处

### Q2: 为什么需要euler2mat转换？

**A:** 
- **欧拉角**直观但有万向锁问题
- **旋转矩阵**是标准数学表示
- **ikpy库**要求输入旋转矩阵
- **MuJoCo**内部使用四元数和矩阵

### Q3: start_joints是如何确定的？

**A:** 
1. 通过正运动学计算（给定关节角→末端位姿）
2. 或通过MuJoCo可视化工具手动调整
3. 或通过IK求解器反向计算
4. 确保机械臂在工作空间内且无碰撞

### Q4: 为什么A和B位置X、Z相同？

**A:** 
- **X相同**：保持在一条直线上，简化轨迹
- **Z相同**：水平推动，不抬高或降低
- **只改变Y**：沿着一个方向直线运动

### Q5: ref_pos的虚拟链节是什么？

**A:** 
- URDF中的`global_transform`固定关节创建虚拟链节
- 用于坐标系对齐（180°旋转）
- ikpy库读取URDF时包含所有链节
- `active_links_mask=[False, False, True*6, False]`标记哪些可控
