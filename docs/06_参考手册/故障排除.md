## 🛠️ 环境配置问题

### 问题1: Qt platform plugin "xcb" 错误

**错误信息：**
```
qt.qpa.plugin: Could not load the Qt platform plugin "xcb" in "" even though it was found.
This application failed to start because no Qt platform plugin could be initialized.
```

**原因：** 缺少Qt xcb平台插件的系统依赖

**解决方法：**
```bash
# 方法1: 使用提供的安装脚本
cd docs/01_环境配置
chmod +x install_qt_deps.sh
./install_qt_deps.sh

# 方法2: 手动安装依赖
sudo apt update
sudo apt install -y \
    libxcb-xinerama0 \
    libxcb-cursor0 \
    libxkbcommon-x11-0 \
    libxcb-icccm4 \
    libxcb-image0 \
    libxcb-keysyms1 \
    libxcb-randr0 \
    libxcb-render-util0 \
    libxcb-shape0
```

---

### 问题2: OpenGL相关错误

**错误信息：**
```
libGL error: failed to load driver: swrast
libGL error: MESA-LOADER: failed to open swrast
```

**原因：** WSL2环境下OpenGL配置不正确

**解决方法（WSL2）：**
```bash
# 取消间接渲染
unset LIBGL_ALWAYS_INDIRECT

# 设置正确的DISPLAY变量
export DISPLAY=:0

# 或使用提供的运行脚本
cd docs/01_环境配置
./run_with_wslg.sh
```

---

### 问题3: 找不到mujoco模块

**错误信息：**
```
ModuleNotFoundError: No module named 'mujoco'
```

**原因：** MuJoCo未安装或虚拟环境未激活

**解决方法：**
```bash
# 检查是否安装
pip list | grep mujoco

# 如果未安装
pip install mujoco>=3.3.0

# 如果使用虚拟环境，确保已激活
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate     # Windows
```

### 问题7: IK求解失败

**错误信息：**
```
警告: IK solver did not converge
```

**原因：** 目标位置超出机械臂工作空间

**解决方法：**
```python
# 检查目标位置是否合理
target_A_pos = [-0.13, 0.5, 0.1]   # 确保在工作空间内

# UR5e工作空间（大致范围）
# X: -0.85 ~ 0.85 米
# Y: -0.85 ~ 0.85 米
# Z: 0.0 ~ 1.2 米

# 如果目标超出范围，调整到安全区域
target_A_pos = [-0.13, 0.5, 0.15]  # 示例修改
```

## 🖥️ 显示问题

### 问题8: MuJoCo窗口无法打开

**症状：** 程序运行但看不到3D窗口

**检查清单：**
```bash
# 1. 检查WSLg是否运行（WSL2环境）
ps aux | grep Xwayland

# 2. 检查DISPLAY变量
echo $DISPLAY
# 应该显示：:0

# 3. 测试X11转发
xclock  # 应该弹出时钟窗口

# 4. 如果都不行，重启WSL
wsl --shutdown
# 重新打开WSL终端
```

---

### 问题9: 窗口打开但看不到机械臂

**原因：** 相机视角问题

**解决方法：**
```python
# 方法1: 在viewer_init()中调整相机
def viewer_init(viewer):
    viewer.cam.lookat[:] = [0, 0.5, 0.5]  # 注视点
    viewer.cam.distance = 1.5              # 拉近距离
    viewer.cam.elevation = -45             # 调整角度

# 方法2: 运行时手动调整
# - 鼠标左键拖拽：旋转视角
# - 鼠标滚轮：缩放
# - 鼠标右键拖拽：平移
```


## 🎮 运动控制问题

### 问题12: 机械臂不移动

**可能原因及解决方法：**

#### 原因1: IK求解失败
```python
# 添加调试信息
print("目标A关节角:", target_A_joints)
print("目标B关节角:", target_B_joints)

# 检查是否全为NaN或无穷大
import numpy as np
if np.any(np.isnan(target_A_joints)):
    print("警告：IK求解失败，包含NaN值")
```

#### 原因2: 控制器增益过低
```xml
<!-- 在ur5e.xml中增大增益 -->
<general gainprm="3000" biasprm="0 -3000 -600" ... />
```

#### 原因3: 目标与起始位置相同
```python
# 检查位置是否有变化
print("初始末端位置:", initial_pos)
print("目标A位置:", target_A_pos)
```

---

### 问题13: 机械臂运动不平滑

**原因：** 轨迹步数太少

**解决方法：**
```python
# 增加轨迹步数
trajectory_to_A = JointSpaceTrajectory(start_joints, target_A_joints, steps=200)
trajectory_to_B = JointSpaceTrajectory(data.qpos[:6], target_B_joints, steps=300)
```

---

### 问题14: 阶段切换不发生

**原因：** 到达检测容差太严格

**解决方法：**
```python
# 放宽容差
if np.allclose(data.qpos[:6], target_A_joints, atol=0.05):  # 从0.02改为0.05

# 或添加超时机制
phase_start_time = data.time
if data.time - phase_start_time > 5.0:  # 5秒超时
    print("超时，强制切换到下一阶段")
    current_phase = "to_B"
```

---

### 问题15: 立方体没有被推动

**可能原因及解决方法：**

#### 原因1: B位置不够远
```python
# 增加推动距离
target_B_pos = [-0.13, 0.70, 0.1]  # 从0.65改为0.70
```

#### 原因2: 末端传感器未接触
```python
# 检查接触力
filtered_force = force_sensor.filter()
force_magnitude = np.linalg.norm(filtered_force)
print(f"当前接触力: {force_magnitude:.2f} N")

# 如果力始终为0，说明未接触
```

#### 原因3: 立方体太重
```xml
<!-- 在scene.xml中减小质量 -->
<geom mass="0.5" ... />  <!-- 从1.0改为0.5 -->
```

## ⚡ 性能问题

### 问题16: 仿真运行很慢

**优化方法：**

#### 方法1: 降低可视化更新频率
```python
force_plotter = ForcePlotter(update_interval=100)  # 更新间隔加大
```

#### 方法2: 增大时间步长
```xml
<!-- 在scene.xml中 -->
<option timestep="0.005"/>  <!-- 从0.002改为0.005 -->
```

#### 方法3: 简化碰撞几何
```xml
<!-- 使用更简单的碰撞体 -->
<geom type="capsule" ... />  <!-- 代替复杂网格 -->
```

## 📋 诊断检查清单

遇到问题时，按顺序检查：

### 基础检查
- [ ] Python版本 >= 3.8
- [ ] 所有依赖包已安装
- [ ] 当前目录正确（项目根目录）
- [ ] 所有必需文件存在

### 环境检查
- [ ] WSL2版本支持WSLg（如使用WSL）
- [ ] DISPLAY环境变量正确
- [ ] 系统依赖完整

### 代码检查
- [ ] 目标位置在工作空间内
- [ ] 关节角度在限位范围内
- [ ] 轨迹步数 > 0
- [ ] 没有语法错误

### 运行时检查
- [ ] 窗口能正常打开
- [ ] 机械臂模型可见
- [ ] 控制台无错误信息
- [ ] 力传感器有读数（如果接触）
